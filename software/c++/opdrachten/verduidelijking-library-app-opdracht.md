Deze opdracht werkt als volgt:
1.	Je kiest een chip waar je een zelf een library voor wilt schrijven. Je downloadt alvast de datasheet om een beeld te krijgen van wat daarbij komt kijken.
2.	Je bedenkt een coole of nuttige applicatie waarin je het toepast. Bij het geval van een temperatuursensor zou het leuk zijn als de applicatie meer doet doet dan alleen temperaturen weergeven. Misschien kan het nog een nuttige - toegepaste setting krijgen? wat doe je met al die temperaturen? NB: je hoeft alleen voor de temperatuursensor zelf de library te schrijven. De overige libraries mag je vrij downloaden en gebruiken.
3.	Het bovenstaande plan beargumenteer je en lever je hier in.
4.	Na feedback van een docent er op kun je er mee aan de slag.
5.	Je begint dan met het zelf schrijven van de library. Je mag daarbij uitgaan van vanilla arduino ide - dus je mag evt bijvoorbeeld wire library gebruiken. Maar geen externe gedownloade libraries. De library schrijf vanuit de datasheet van de chip (vanaf scratch - zonder te spieken in de externe door anderen geschreven library - het is nu juist de bedoeling dat je zelf laat zien dat je vanaf een datasheet van een chip vanaf scratch een libary kunt schrijven). Houdt daarbij een logboek bij waarin je je gedachtengangen noteert, en ook de elementen uit de datasheet die je belangrijk vindt om mee te nemen in je library. Je probeert zoveel mogelijk features van de chip te ondersteunen. Als je ze niet allemaal ondersteunt, even aangeven om welke het gaat en beargumenteren.
6.	Wat basis tests schrijven en een applicatie die een leuke, wat uitgebreidere toepassing laat zien.
7.	Helemaal mooi en aanbevolen is als je library via een slim gekozen abstracte interface werkt, zodat de temperatuursensor in de toekomst eventueel makkelijk door een andere kan worden vervangen zonder dat je de code van je applicatie hoeft aan te passen.
